# Component — UI Component Design Reference

Quick-reference skill for component architecture, composition patterns, and design system fundamentals.

## When to Use

- Designing reusable UI component APIs
- Choosing between composition patterns (render props, slots, HOC)
- Defining prop interfaces and validation
- Implementing component lifecycle correctly
- Building accessible, testable component libraries

## Commands

### `intro`

```bash
scripts/script.sh intro
```

Overview of component-based architecture — principles and mental model.

### `propsapi`

```bash
scripts/script.sh propsapi
```

Props API design: naming, types, defaults, required vs optional, validation.

### `composition`

```bash
scripts/script.sh composition
```

Composition patterns: children, slots, render props, compound components, HOC.

### `lifecycle`

```bash
scripts/script.sh lifecycle
```

Component lifecycle across React, Vue, Svelte — mount, update, unmount.

### `statemgmt`

```bash
scripts/script.sh statemgmt
```

Component state management: local vs lifted vs global, controlled vs uncontrolled.

### `accessibility`

```bash
scripts/script.sh accessibility
```

Accessible component patterns: ARIA roles, keyboard navigation, focus management.

### `testing`

```bash
scripts/script.sh testing
```

Component testing strategies: unit, integration, visual regression, storybook.

### `patterns`

```bash
scripts/script.sh patterns
```

Common component patterns: modal, dropdown, form field, data table, toast.

### `help`

```bash
scripts/script.sh help
```

### `version`

```bash
scripts/script.sh version
```
