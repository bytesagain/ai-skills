# Enum — Enumeration Pattern Reference

Quick-reference skill for enum design patterns, bit flags, and type-safe state modeling.

## When to Use

- Designing enums that are forward-compatible and serializable
- Implementing bit flags for combining multiple options
- Choosing enum patterns across languages (Rust, TS, Python, Java, Go)
- Modeling state machines with enum transitions
- Handling unknown/default enum values safely

## Commands

### `intro`

```bash
scripts/script.sh intro
```

Overview of enumerations — purpose, types, and design principles.

### `languages`

```bash
scripts/script.sh languages
```

Enum syntax and patterns in Rust, TypeScript, Python, Java, C#, Go, C.

### `bitflags`

```bash
scripts/script.sh bitflags
```

Bit flags and bitmasks — combining enums with bitwise operations.

### `statemachine`

```bash
scripts/script.sh statemachine
```

Modeling state machines with enums — transitions, guards, and patterns.

### `serialization`

```bash
scripts/script.sh serialization
```

Enum serialization — string vs int, unknown value handling, schema evolution.

### `algebraic`

```bash
scripts/script.sh algebraic
```

Algebraic data types — Rust-style enums with associated data (sum types).

### `antipatterns`

```bash
scripts/script.sh antipatterns
```

Enum anti-patterns: string abuse, boolean blindness, primitive obsession.

### `database`

```bash
scripts/script.sh database
```

Enums in databases: CHECK constraints, lookup tables, and migration strategies.
