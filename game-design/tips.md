# Game Design Tips

## 设计核心

1. **乐趣至上** — 任何设计决策都要服务于"好玩"
2. **玩家视角** — 想象自己是第一次玩的玩家
3. **快速原型** — 纸面原型测试比写代码更快验证想法
4. **迭代改进** — 好游戏是改出来的，不是一次设计完美的

## 核心循环设计

**短循环（秒级）**：一次操作的即时反馈
**中循环（分钟级）**：一局/一关的体验
**长循环（小时级）**：整体成长和目标感

## 经济系统平衡

| 要素     | 注意事项                      |
|---------|------------------------------|
| 货币获取  | 控制产出速度，避免通胀          |
| 货币消耗  | 提供足够的消费坑，保持流通      |
| 稀缺性   | 核心资源要有一定稀缺感          |
| 公平性   | 付费玩家与免费玩家的体验平衡    |

## 常见玩法模式

- **收集**：收集要素驱动探索
- **成长**：数值成长带来成就感
- **竞技**：与他人对抗的刺激感
- **创造**：自由建造的表达欲
- **社交**：与人互动的归属感
- **叙事**：沉浸式故事体验

## 新手引导

- 前5分钟决定玩家去留
- 边玩边学，不要文字轰炸
- 确保第一次成功体验
- 逐步解锁复杂功能
