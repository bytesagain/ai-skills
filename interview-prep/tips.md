# 面试全攻略 Tips

## 一、面试前准备 Checklist

### 📋 必做项
- [ ] 研究公司：产品线、近期新闻、融资情况、竞争对手
- [ ] 熟读JD：标记每个要求，准备对应案例
- [ ] 准备自我介绍：1分钟版 + 3分钟版
- [ ] 准备3-5个STAR故事，覆盖不同能力维度
- [ ] 准备反问问题（至少3个）
- [ ] 确认面试形式（线上/线下）、着装要求
- [ ] 测试设备（线上面试：网络、摄像头、麦克风）
- [ ] 打印简历2-3份（线下面试）

### 🧠 加分项
- [ ] 了解面试官背景（LinkedIn）
- [ ] 准备作品集/项目演示
- [ ] 模拟面试练习（找朋友或录像复盘）
- [ ] 了解公司薪资范围和福利体系

## 二、常见30题 + 参考答案框架

### HR轮（10题）

1. **请自我介绍一下**
   → 公式：当前身份 + 核心成就 + 为什么来面试

2. **为什么离开上一家公司？**
   → 原则：说发展，不说不满；说未来，不说过去

3. **为什么选择我们公司？**
   → 公式：行业认可 + 产品/文化欣赏 + 个人匹配

4. **你的职业规划是什么？**
   → 原则：短期（1-2年）具体，长期（3-5年）方向性

5. **期望薪资是多少？**
   → 策略：给范围不给数字，先了解预算

6. **你最大的优点是什么？**
   → 原则：与岗位相关 + 有具体例子

7. **你最大的缺点是什么？**
   → 策略：真实但不致命 + 正在改进的措施

8. **你如何处理工作压力？**
   → 公式：承认压力存在 + 具体应对方法 + 实际案例

9. **你有什么要问我的吗？**
   → 策略：问团队、问发展、问挑战，不问假期和加班

10. **多久能入职？**
    → 原则：给合理时间，展现积极态度

### 技术轮（10题）

11. **介绍一个你最有成就感的项目**
    → STAR法则：情境 + 任务 + 行动 + 结果

12. **遇到过最大的技术难题是什么？**
    → 重点：问题定义 + 解决过程 + 学到什么

13. **你怎么保持技术更新？**
    → 列举：博客/开源/社区/实践项目

14. **如果让你重做这个项目，你会怎么改？**
    → 展现：反思能力 + 技术深度

15. **你怎么看待代码质量？**
    → 维度：可读性、测试、文档、code review

16. **估算/系统设计类问题**
    → 方法：明确需求 → 拆解问题 → 计算/设计 → 验证

17. **你对我们产品有什么建议？**
    → 准备：提前体验产品，找1-2个改进点

18. **你怎么和非技术人员沟通？**
    → 关键：类比、可视化、避免术语

19. **描述你debug的过程**
    → 展现：系统性思维，不是靠猜

20. **你觉得好的架构应该是怎样的？**
    → 关键词：可扩展、可维护、高可用、简单

### 终面（10题）

21. **你怎么理解我们的业务？**
    → 展现：做了功课，有自己的见解

22. **如果团队意见不一致，你怎么处理？**
    → 原则：倾听 → 分析 → 决策 → 执行

23. **你怎么带领团队？（管理岗）**
    → 维度：目标设定、人才培养、绩效管理

24. **你犯过最大的错误是什么？**
    → 重点：错误本身不重要，复盘和改进才重要

25. **公司正在面临XX挑战，你怎么看？**
    → 展现：战略思维、商业敏感度

26. **你期望在这里实现什么？**
    → 平衡：个人成长 + 对团队/公司的贡献

27. **你怎么看待加班？**
    → 策略：承认必要性，但强调效率优先

28. **你的同事会怎么评价你？**
    → 技巧：引用真实反馈，不要自己编

29. **如果给你100万预算，你会做什么项目？**
    → 展现：商业思维 + 创新能力

30. **你还有其他offer吗？**
    → 策略：适度透露增加紧迫感，不要虚构

## 三、谈薪技巧

### 核心原则
1. **永远不要先报价** — 让对方先出牌
2. **报范围不报数字** — "我期望在25-30K之间"
3. **谈总包不谈月薪** — 基本工资 + 奖金 + 股票 + 福利
4. **有备选才有底气** — 手上有其他offer是最强谈判筹码

### 话术模板

**被问期望薪资时：**
> "基于我的经验和市场行情，我期望的范围是XX-XX。但我更看重整体的发展机会，如果岗位匹配度高，薪资方面我们可以进一步沟通。"

**对方报价偏低时：**
> "感谢offer，我对这个岗位非常感兴趣。不过基于我在XX方面的经验，以及目前市场上类似岗位的薪资水平，我期望能在XX左右。您看是否有调整空间？"

**多个offer对比时：**
> "目前我手上有另一个offer，在薪资上比较有竞争力。但我对贵司的XX方向更感兴趣，如果薪资能到XX，我会优先选择贵司。"

### 谈判节奏
1. 拿到offer → 表达感谢和兴趣
2. 确认完整薪资包（不要只看月薪）
3. 如需谈判 → 给出合理依据
4. 达成一致 → 邮件确认所有条款
5. 签正式offer → 确认入职日期

## 四、各轮面试重点

### HR轮
- **考察重点**：文化匹配、沟通能力、稳定性、动机
- **常见坑**：问离职原因时说前公司坏话
- **技巧**：展现真诚和热情，但保持职业感

### 技术轮
- **考察重点**：专业能力、问题解决、技术深度
- **常见坑**：不懂装懂、只说结论不说过程
- **技巧**：思考过程比答案更重要，遇到不会的坦诚说

### 终面/高管面
- **考察重点**：战略思维、领导力、文化认同
- **常见坑**：格局太小，只谈执行不谈思考
- **技巧**：站在业务角度思考，展现大局观

## 五、STAR法则速查

```
S - Situation（情境）: 什么背景？什么时候？
T - Task（任务）: 你的目标/职责是什么？
A - Action（行动）: 你具体做了什么？怎么做的？
R - Result（结果）: 产生了什么成果？有数据吗？
```

### 好的STAR回答示例

> **S**: 去年Q3，我们产品的用户留存率从45%持续下滑到32%。
> **T**: 作为产品负责人，我需要在两个月内扭转这个趋势。
> **A**: 我组织了用户调研（100+用户），发现核心问题是新手引导太复杂。于是重新设计了onboarding流程，将步骤从12步缩减到5步，并加入了个性化推荐。
> **R**: 改版上线后，新用户7日留存从32%提升到51%，DAU也增长了18%。

## 六、面试后跟进时间线

| 时间点 | 动作 |
|--------|------|
| 面试当天 | 发感谢信/邮件 |
| 3个工作日后 | 如无回复，礼貌跟进 |
| 1周后 | 再次跟进，表达持续兴趣 |
| 2周后 | 最后一次跟进，确认结果 |
| 超过2周无回复 | 继续投递其他机会 |
