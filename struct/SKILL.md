# Struct — Composite Data Type Reference

Quick-reference skill for struct/record types — definition, memory layout, methods, and cross-language patterns.

## When to Use

- Defining composite data types with named fields
- Understanding struct memory layout and alignment
- Comparing struct vs class semantics
- Implementing methods on structs
- Serializing/deserializing structured data

## Commands

### `intro`

```bash
scripts/script.sh intro
```

What structs are, struct vs class, value vs reference semantics.

### `clang`

```bash
scripts/script.sh clang
```

C structs — definition, typedef, nested structs, designated initializers.

### `golang`

```bash
scripts/script.sh golang
```

Go structs — fields, embedding, tags, methods with receivers.

### `rust`

```bash
scripts/script.sh rust
```

Rust structs — named, tuple, unit structs, impl blocks, derive macros.

### `memory`

```bash
scripts/script.sh memory
```

Memory layout — alignment, padding, size, packing, cache optimization.

### `patterns`

```bash
scripts/script.sh patterns
```

Design patterns: builder, newtype, phantom types, opaque structs.

### `serialization`

```bash
scripts/script.sh serialization
```

Struct serialization: JSON, binary, protobuf, zero-copy.

### `comparison`

```bash
scripts/script.sh comparison
```

Struct features compared across C, Go, Rust, Swift, TypeScript, Python.

### `help`

```bash
scripts/script.sh help
```

### `version`

```bash
scripts/script.sh version
```
